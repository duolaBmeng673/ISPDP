# Secure Auth Demo

这是一个适合课程演示的“安全注册/登录”示例项目：

- 前端：原生 HTML + CSS + JavaScript
- 后端：Node.js + Express
- 数据库：MySQL
- 安全特性：Argon2id 密码哈希、RSA 双因子挑战签名、邮箱验证码跨设备恢复、HttpOnly Session Cookie、限流、输入校验、Helmet 安全头
- 业务界面：商品信息列表、购物车、订单确认、模拟付款确认

## 运行方式

0. 执行：   
    `mysql -h 127.0.0.1 -P 3306 -u root -p ispdp < schema.sql`  
    `mysql -h 127.0.0.1 -P 3306 -u root -p ispdp < seed.sql`    
    就能恢复数据库结构和演示数据。
1. 安装依赖：`npm install`
2. 复制环境变量：将 `.env.example` 复制为 `.env`
3. 修改 `.env` 中的 `SESSION_SECRET` 和 MySQL 连接信息
4. 启动项目：`npm run dev`
5. 浏览器访问：`http://localhost:3000`

## 核心流程

1. 注册时浏览器使用 `Web Crypto API` 本地生成 RSA 密钥对
2. 服务端只接收公钥，私钥保存在浏览器 `IndexedDB`
3. 登录第一步先验证用户名和密码
4. 密码正确后，服务端下发随机挑战码
5. 浏览器使用本地 RSA 私钥对挑战码签名
6. 服务端使用数据库中的公钥验签
7. 验签成功后才创建最终登录会话
8. 如果用户更换设备，可通过绑定邮箱接收验证码并重新绑定新的 RSA 公钥
9. 登录后进入商品销售页面，可完成商品浏览、加入购物车、提交订单和模拟付款确认

## 课程答辩可以强调的点

- 密码不会明文保存在数据库中，而是使用 `Argon2id` 哈希存储
- 登录时引入 RSA 挑战签名，证明用户不仅知道密码，还持有本地私钥
- 当用户更换设备或丢失私钥时，可通过绑定邮箱完成安全恢复并重绑新设备
- 服务端只保存公钥和挑战摘要，不保存私钥，不明文存储挑战码
- 登录态通过 `HttpOnly` Cookie 保存，避免前端脚本直接读取敏感会话信息
- 使用 `helmet` 添加常见安全响应头
- 对登录和注册接口增加限流，降低暴力破解风险
- 前后端都进行了输入校验，减少非法数据进入系统
- 启动时会自动确保 MySQL 中存在 `ispdp` schema、`users`、`login_challenges` 和 `email_verification_codes` 表
- 启动时会自动确保 MySQL 中存在 `products`、`orders`、`order_items` 业务表，并自动写入演示商品

## 注意

- 真正部署到线上时，必须启用 HTTPS，这样密码传输过程才会受到 TLS 保护
- 当前项目默认连接本机 MySQL；如果数据库地址变化，只需修改 `.env`
- 生产环境请配置真实 SMTP 参数，否则验证码邮件只会输出到服务器日志用于本地演示
- RSA 私钥与浏览器环境绑定，如果用户清空站点数据，可通过邮箱恢复重新绑定
